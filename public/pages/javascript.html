Variables

Variables in JavaScript can be defined with const, let and var. Looking at history, var was the primary way to define a function until let and const was implemented in JavaScript.

const variable1 = 'Hello'
let variable2 = 'from'
var variable3 = 'JavaScript'

const declared variables cannot be reassigned. This means that if we instantiate a const variable, it is not possible to reassign the value of that variable.
Example:

const variable1 = 'Hello'
variable1 = 'newValue'
Error: Uncaught TypeError: Assignment to constant variable.

Luckily JavaScript has let which makes it possible to reassign the variable
Example:

let variable1 = 'Hello'
variable1 = 'newValue'
// Outputs: 'newValue'

Let's now make a function to withdraw money to from wallet. Our function should look like this and make use of let:

// We need a varialbe to keep track of the total amount in the wallet
let totalWallet = 100;
// And now the function
function deductAmount(amount) {
// If total is more or equal to amount, we can deduct
if(totalWallet >= amount) {
totalWallet -= amount;
} else {
// If not, not enough money is in the wallet
console.log('Not enough money');
}
}

Now this makes it easy to reassign the value of a let variable, as this wouldn't be possible using const.
But now what is var really and how can we make use of it?
Let's take an example with the same code block as before:

let totalWallet = 100;
function deductAmount(amount) {
if(totalWallet >= amount) {
// Create a new var variable inside the if scope
var newWallet = totalWallet -= amount;
} else {
// Set the var to the original total of the wallet
newWallet = totalWallet
console.log('Not enough money');
// Return the var variable
}
return newWallet;
}

With var it's possible to declare and instantiate a new variable inside a scope and then return it outside of the scope. This would not be possible with either const or let.
A best practice is to only use const or let as it is easier to troubleshoot and makes the code more readable.

JavaScript has 8 primitive data types:

  • - null
  • - undefined
  • - boolean
  • - nnumber
  • - bigint
  • - string
  • - symbol

A primitve data type is defined as data that has no properties or methods. width the typeof keyword It's possible to console.log the data type of a variable. This is relevant because numbers can be strings and vice versa

const numberOne = 1;
const numberTwo = '2';
// Logs 'number' as it isn't wrapped in quotes
console.log(typeof numberOne);
// Logs 'string' as it is wrapped in quotes
console.log(typeof numberOne);
// Logs '12' as it is both a number and an string
// JavaScript displays it as a string
console.log(numberOne + numberTwo);
// If we want to actually add them
// we would have to cast the string to a number
const newNumberTwo = Number(numberTwo);
// Logs 3 as it is both variables are numbers
console.log(numberOne + newNumberTwo);

Some of the primitive data types can actually be objects using an object wrapper. Though we want primitives to be fast and lightweight, the object wrapper gives functionality to actually use functions on them. This means that we can use methods/functions to capitalize strings as an example.

const myString = 'hello';
const upperCaseString = myString.toUpperCase();
// Outputs 'HELLO'
console.log(upperCaseString);

Objects

Objects data types are arrays and objects. We can define an object by using const, let or var and then wrapping it's properties in curly brackets.

const person = {
 name: 'Tom',
 lastName: 'Cruise',
 age: 61
}

To access these attributes such as name an age in this example, we can call the name of the variable following by a dot an then the name of the property.

// Outputs 'Tom'
console.log(person.name);

We can also assign the properties to other values, and this is a key part of const. The variable itself cannot be reassigned but it's properties can.

// Reassign the property name
person.name = 'Thomas'
// Outputs 'Thomas'
console.log(person.name)

Arrays can be defined by using const, let or var following by a set of square brackets. Arrays can contain objects aswell as primitves and are accessed through index or other methods defined in the arrays class such as push(), pop() and slice()

const numbers = [1, 2, 4, 6, 7, 10]
// Outputs 1
console.log(numbers[0])

// Pushes an element to the end of an array
numbers.push(12)
// numbers is now [1, 2, 4, 6, 7, 10, 12]

// Pop an element (remove from the top)
numbers.pop()
// numbers is now back to [1, 2, 4, 6, 7, 10]

Functions

Functions can be defined like this:

function multiply(a,b) {
return a*b;
}

This function returns the two numbers a and b multiplied

const result = multiply(2,3)
// Outputs 6
console.log(result)

Functions can also be defined with arrows and can be either assigned to variables or be anonymous:

// Example anonymous functions
const multiply3 = function (a*b) {
return a*b
}

// Example of arrow functions
const multiply = (a,b) => a*b;

const multiply2 = (a,b) => {
return a*b;
}

Callbacks

Callback functions are used to handle JavaScript's asynchronous way of handling things. If we make a call to fetch some data somewhere it's important to now that all data has been retrieved before updating the UI as an example. In other words we use callbacks to execute some code after we are sure something else has been executed before the callback.
A callback function is defined by passing a function as a parameter to another function.
Some callback functions are build into JavaScript such as setTimeout() or listening to events with addEventlistener().

// Example of a callback
function greet(name, callback) {
console.log(`hello ${name}`);
callback();
}

function theCallback() {
console.log('nice to meet you');
}

greet('Tom Cruise', theCallback);

Promises

Another way of handling asynchronous JavaScript is using promises.
A promise has three possible states: pending, fulfilled or rejected. We use a promise by adding a .then() after a function call. When the promise is is created it has a pending state. Then it's either fulfilled or rejected. But before we know if its fulfilled or rejected, we chain another .then() to get the actual data from the call.
Adding a .catch() in the end of a promise catches any error that may occur.
In this example fetch() will be used, which makes it possible to get data from an API but always returns a promise.

// Example of a promise
fetch('/api/data')
.then(response => response.json())
.then(data) => {
console.log(data)
})
.catch(error => {
console.log(error);
})

Async / Await

Is also possible to hande asynchronous code using async and await, which is written in fewer lines than a promise, but they serve the same purpose. The syntax is a bit different. Add a async keyword the in the function declaration and use await before calling the function with await.
One downside to using async and await is to have it wrapped in a try catch block. This is the only way we can handle errors.

try {
const response = fetch('api/data')

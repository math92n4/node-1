Create first node project

Node is created to write server side JavaScript, and is installed by visiting Node.js download.
To get started with the first node.js project open up a desired folder and create a package.json file containing:

{
"dependencies": {
}
}

Open a terminal and go to the folder of package.json file and type:

npm install express

This creates the node_modules folder and installs a new dependency in the package.json file. Express will be covered later.
If we ever want to execute a file within a node enviroment, we can open a console, find the folder where our JavaScript is located and run node app.js (assuming our js file is called app)

node app.js

node_modules

node_modules is a folder that contains all dependencies a node project uses. So its quite a big directory. When npm is used (node package manager) given the following command is a known dependency, the dependency is downloaded to the node_modules folder. Like in the express example before. In a node project, it's possible to import these modules and use them in the project's code. Lets try to import the fs which is a module read files. Write this at the top of a file:

import fs from "fs"
// Lets read a file using fs
const file = fs.readFileSync('text.txt', 'utf-8');
// The file constains 'Tom Cruise'
console.log(file)
// Outputs: Tom Cruise

Keep in mind that the second parameter in fs.readFileSync() can be swapped with a toString(). This would be the same result

// Same results
const file = fs.readFileSync('text.txt', 'utf-8');
const fileToString = fs.readFileSync('text.txt').toString();

.json

The .json format stands for JavaScript Object Notation and is probably the most used format for sending data around from different APIs. This is one way using json, but in node we also use this format to declare different dependencies. Just like maven uses xml and other dependency / build tools.
A .json file always starts with an empty set of curly brackets. Inside these are the data.
Arrays are defined with square brackets and objects with curly brackets. Almost just like normal JavaScript. The main difference is putting property names / keys in quotes.
Within our package.json we can define other things such as scripts and devDependencies.

// JSON
"name": "Tom",
"lastname": "Cruise"

// JavaScript
name: "Tom",
lastname: "Cruise"

Nodemon

Nodemon is a utility from node.js that allows you to restart an app when changes to files or new files has been created.
This is very useful if a server is running on your node app. Instead opening a console and type node app.js every time a change has been made, we can simply type:

npx nodemon app.js

In fact, we can create a script in our package.json that runs this command for us.

// package.json
{
"dependencies": {
"express": "^4.18.3"
},
"scripts": {
"dev": "npx nodemon app.js
}
}

This allows us to enter the terminal a write npm run dev as our script is called dev. This makes the app restart on any change.
So lets imagine a very large app with a lot of files. In some cases it's useful to define what type of files nodemon should listen to if changes are made, so we can actually make the script even smarter:

// package.json
"scripts": {
"dev": "nodemon --ext html,js,json app.js"
}

This dev script now only listens to html, js or json files. Also it listens to a specific file called app.js, where our server is defined.
